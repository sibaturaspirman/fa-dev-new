User-friendly

Pencatatan mudah tanpa program

Simpan data yang dikumpulkan dalam format CSV pada kartu memori SD hanya dengan menyelesaikan pengaturan mudah dengan wizard alat pengaturan khusus. Berbagai materi referensi termasuk laporan harian, pembuatan formulir, dan laporan umum dapat dibuat dengan mudah dalam berkas CSV yang tersimpan. Data ini dapat digunakan untuk berbagai macam aplikasi yang memerlukan keterlacakan, data produksi, dll.

■ Pengaturan dengan layar Wizard

Pencatatan varians data kontrol

Data dikumpulkan selama setiap pemindaian atau dalam interval milidetik yang memungkinkan deteksi deviasi kontrol bahkan pada kecepatan yang sangat tinggi. Oleh karena itu, identifikasi kesalahan dapat dilakukan lebih cepat dan lebih rinci.

■ Data sampel generik dari komputer atau perangkat eksternal pada interval 100 ms

■ Fungsi pencatatan data CPU Seri Q mampu mengambil sampel data pada interval yang jauh lebih tinggi untuk mendeteksi nilai yang berubah dengan cepat.

Pencatatan data otomatis hanya dengan menggunakan kartu memori SD

Pencatatan data otomatis dilakukan hanya dengan memasukkan kartu memori SD ke dalam CPU, yang dicapai karena kartu memori menyertakan berkas konfigurasi pencatatan. Instruksi pencatatan data dari jarak jauh juga dilakukan hanya dengan mengirimkan berkas konfigurasi melalui email dan menyalinnya ke kartu memori SD.

Kirim file pencatatan ke server FTP secara otomatis

File pencatatan data yang disimpan di kartu memori SD dapat dikirim ke server FTP hanya dengan melakukan pengaturan sederhana menggunakan alat konfigurasi Pencatatan.
Karena server pencatatan dapat menangani banyak file, tugas pengelolaan dan pemeliharaan dapat dikurangi.

Ramah Pengguna

Terhubung dengan mudah ke CPU melalui Ethernet

Pengaturan alamat IP tidak diperlukan untuk terhubung langsung ke modul CPU (koneksi satu-ke-satu) menggunakan GX Works2 atau GX Developer. Kabel lurus dan kabel silang dapat digunakan, dan diidentifikasi secara otomatis oleh modul CPU. Oleh karena itu, metode koneksi ini semudah menggunakan USB. Bahkan operator yang tidak terbiasa dengan pengaturan jaringan dapat dengan mudah membuat koneksi.

CC-Link IE Field Network Basic tidak memerlukan modul jaringan

CPU pengontrol terprogram dengan port Ethernet tertanam dapat digunakan sebagai stasiun induk*1, sehingga menghilangkan kebutuhan akan modul jaringan tambahan. Jaringan dapat dikonfigurasi dengan jumlah modul minimum yang mengurangi biaya ruang dan perangkat keras.

  • *1.Hanya didukung oleh QCPU model Universal berkecepatan tinggi dan CPU proses model Universal.

  • *3.Hanya didukung oleh QCPU model Universal berkecepatan tinggi dan CPU proses model Universal.

Komunikasi yang lancar di semua lapisan

QCPU model Universal mendukung banyak teknologi jaringan termasuk CC-Link IE Controller Network berkecepatan tinggi dan berkapasitas tinggi serta CC-Link IE Field Network. Bersama dengan MELSECNET/H, Ethernet, dan CC-Link, jaringan ini dapat diakses dengan lancar di luar jenis atau hierarki jaringan. Setiap pengontrol yang dapat diprogram pada jaringan dapat diakses untuk tugas pemrograman dan pemeliharaan dengan menggunakan komputer pribadi dengan peralatan teknik yang sesuai yang terhubung melalui Ethernet.

Data jam yang akurat

Jam modul CPU secara otomatis dikoreksi dengan fungsi sinkronisasi jam SNTP*4. Ketika data jam CPU disinkronkan secara andal antara sistem, setiap kejadian atau kesalahan bercap waktu yang melibatkan lebih dari satu CPU dapat dengan mudah dipahami dalam hal urutan kejadian dan hubungannya.

Fungsi pelacakan sampel<sup>*8</sup>

Fungsi pelacakan sampel adalah alat diagnostik yang berguna untuk menganalisis data kesalahan, dan urutan kejadian untuk debug program, dll. Fungsi ini dapat membantu mengurangi waktu keseluruhan yang diperlukan untuk memulai dan mengoperasikan peralatan.
Dalam konfigurasi beberapa CPU, fungsi ini dapat membantu menentukan waktu dan transfer data antara modul CPU. Data yang terkumpul dapat dengan mudah dianalisis dalam perangkat lunak pemrograman dengan perbedaan nilai perangkat kata dan perangkat bit yang ditampilkan dengan mudah dalam bentuk bagan dan grafik.
Selain itu, hasil dari jejak pengambilan sampel dapat diekspor ke format file CSV GX LogViewer untuk analisis dalam perangkat lunak.

Proses debugging

CPU model universal memiliki kemampuan untuk menggunakan fungsi "Executional conditional device test", yang secara otomatis menetapkan nilai perangkat ke nilai yang ditentukan pengguna pada setiap langkah selama simulasi program. Secara tradisional, untuk mensimulasikan I/O nyata atau perubahan nilai perangkat lainnya, program terpisah perlu ditulis untuk melakukan debugging. Dengan menggunakan fungsi "Executional conditional device test", dimungkinkan untuk men-debug bahkan sebagian kecil dari program ladder sederhana tanpa perlu memodifikasi program atau menambahkan anak tangga ladder. Oleh karena itu, debugging dapat diselesaikan dengan lebih cepat dan mudah.

Penetapan titik perangkat

Jangkauan perangkat bit yang diperluas
Perangkat bit, relai internal (M) dan relai tautan (B), kini dapat ditetapkan hingga 60K titik masing-masing. Model sebelumnya dibatasi hingga 32K titik.
Jumlah total titik perangkat tetap sama, namun fleksibilitas yang lebih besar dalam pemanfaatan dan pemrograman perangkat tercapai.

Pengaturan perluasan register berkas: register data dan register tautan*9
Jumlah titik perangkat Register Data (D) dan Register Tautan (W) dapat diperluas menggunakan ROM standar atau kartu memori. Model sebelumnya hanya memungkinkan perluasan titik perangkat Register Berkas (R/ZR). Dengan menggunakan pengaturan ini, mudah untuk membuat lebih banyak data atau menghubungkan register untuk mengakomodasi perubahan program, dll.

  • *9.Tidak didukung oleh Q00UJ.
