Transmisi siklik

Dalam transmisi siklik, data ditransmisikan secara berkala menggunakan perangkat tautan. Di bawah jaringan CC-Link IE Field dan Controller, terdapat beberapa perbedaan seperti yang dijelaskan di bagian ini.

Jaringan CC-Link IE Field*26

Komunikasi satu-ke-satu dimungkinkan antara stasiun induk dan stasiun perangkat. Status perangkat tautan (RY, RWw) di dalam stasiun induk ditransmisikan ke perangkat eksternal yang terhubung ke stasiun perangkat. Demikian pula, status perangkat eksternal ditransmisikan ke perangkat tautan stasiun induk (RX, RWr) melalui stasiun perangkat.
Dalam kasus stasiun lokal, status perangkat tautan stasiun induk (RY, RWw) diteruskan ke semua perangkat tautan stasiun lokal (RX, RWr) di jaringan. Bila input dari perangkat atau stasiun lokal dijalankan, status perangkat penghubung stasiun perangkat (RX, RWr) dan status perangkat penghubung stasiun lokal (RY, RWw) disimpan dalam perangkat penghubung stasiun induk (RX, RWr), bersama dengan perangkat penghubung stasiun lokal lainnya (RY, RWw). Hasilnya, semua stasiun lokal memiliki data dari stasiun perangkat lainnya, mirip dengan stasiun induk.

  • *26.Data ditransmisikan dengan metode yang sama pada CC-Link IE Field Network Basic. Perhatikan bahwa komunikasi dilakukan antara stasiun induk dan stasiun perangkat saja. Tidak ada stasiun lokal yang tersedia pada CC-Link IE Field Basic.

Keluaran dari stasiun induk

  1. ❶Di stasiun induk, perangkat modul CPU menyala.
  2. ❷Di stasiun induk, status perangkat modul CPU disimpan dalam perangkat tautan (RY, RWw) dengan penyegaran tautan.
  3. ❸Status perangkat tautan stasiun induk (RY, RWw) kemudian disimpan dalam perangkat tautan stasiun perangkat (RY, RWw), dan dalam perangkat penghubung stasiun lokal (RX, RWr) dengan pemindaian tautan.
  4. ❹Status perangkat penghubung stasiun lokal (RX, RWr) disimpan dalam perangkat modul CPU.

Input dari perangkat atau stasiun lokal

  1. ❺Di stasiun lokal, perangkat modul CPU menyala.
  2. ❻Di stasiun lokal, status perangkat modul CPU adalah disimpan dalam area transmisi perangkat tautan stasiunnya sendiri (RY, RWw).
  3. ❼Status perangkat tautan stasiun (RX, RWr), dan perangkat tautan stasiun lokal (RY, RWw) disimpan dalam perangkat tautan stasiun induk (RX, RWr) melalui pemindaian tautan.
  4. ❽Status perangkat tautan stasiun induk (RX, RWr) disimpan dalam perangkat modul CPU melalui penyegaran tautan.

Jaringan Pengendali CC-Link IE

Komunikasi menggunakan relai tautan (LB) dan register tautan (LW)

Fungsi ini memungkinkan setiap stasiun untuk menulis data ke rentang kirimnya sendiri dari perangkat penghubung (LB, LW) untuk mengirimkannya ke semua stasiun lain di jaringan. Data status perangkat penghubung stasiun kontrol (LB, LW) disimpan dalam perangkat penghubung (LB, LW) dari setiap stasiun normal. Demikian pula, status perangkat penghubung stasiun normal (LB, LW) disimpan dalam perangkat penghubung (LB, LW) dari stasiun kontrol dan stasiun normal lainnya.

  1. ❶Di stasiun transmisi, perangkat modul CPU menyala.
  2. ❷Di stasiun transmisi, status perangkat modul CPU disimpan di perangkat tautan (LB, LW) dari modul pendukung Jaringan Pengendali CC-Link IE melalui penyegaran tautan.
  3. ❸Status perangkat tautan (LB, LW) di stasiun transmisi dikirim ke perangkat tautan (LB, LW) dari modul pendukung Jaringan Pengendali CC-Link IE di stasiun penerima melalui pemindaian tautan.
  4. ❹Di stasiun penerima, status perangkat tautan (LB, LW) disimpan dalam perangkat modul CPU.

Transmisi menggunakan input tautan (LX) dan output tautan (LY)

Stasiun induk I/O, yang mengendalikan input tautan (LX) dan output tautan (LY), dan stasiun lain melakukan komunikasi satu ke satu. LX adalah data input yang ditransmisikan antara stasiun dalam satu blok, dan LY adalah data output yang ditransmisikan dari stasiun induk I/O dalam satu blok. Stasiun kontrol atau normal dapat berupa stasiun induk I/O, dan hingga dua stasiun induk I/O (blok 1 dan blok 2) dapat digunakan per jaringan.

Transmisi sementara

Fungsi ini memungkinkan komunikasi dengan stasiun lain saat permintaan dibuat dengan metode seperti instruksi khusus dan perangkat lunak rekayasa. Komunikasi dengan jaringan yang berbeda juga dimungkinkan.
